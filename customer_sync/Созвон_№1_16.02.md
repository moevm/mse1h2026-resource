# Протокол встречи

## Цель и бизнес-смысл проекта
**Цель проекта** - разработка универсального решения для визуализации распределённых систем на основе данных телеметрии.  

**Решение должно:**
- помогать анализировать архитектуру микросервисных приложений;
- выявлять узкие места и деградации производительности;
- позволять гибко настраивать отображение зависимостей между компонентами.

**Целевая аудитория решения:**
- DevOps-инженеры;
- администраторы инфраструктуры;
- команды разработки на этапе эксплуатации и поддержки.

## Функциональность

**Ключевая функция** - Визуализация метрик и зависимостей системы в виде графа

**Предполагается:**
- отображение всех сущностей системы в виде вершин;
- отображение связей в виде рёбер;
- отображение дополнительной информации об узле;
- возможность выбора конкретной вершины и отображения только связанной с ней части графа;
- возможность включения/отключения отдельных типов связей (фильтры, чекбоксы).

**Примеры сценариев:**
- сервис → downstream-вызовы → база данных / кэш;
- топик → consumer → сервисы → downstream-зависимости.

**Допускается использование открытых источников данных или тестовых выгрузок для построения прототипа.**

**Ключевая задача** - приведение данных из разных источников к единому пространству терминов.

## Источники данных

- **проект ориентирован на универсальность** - предполагается, что инструмент должен работать с различными системами телеметрии;
- разные инструменты могут использовать разные наименования сущностей;
- по смыслу необходимые вершины и связи присутствуют в стандартных наборах метрик;
- при необходимости заказчик может предоставить дополнительные выгрузки.
